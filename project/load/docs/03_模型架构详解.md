# 模型架构详解

## 1. 模型概览

系统使用三类模型进行负荷预测，通过集成策略选择最优输出：

| 模型 | 类型 | 输入 | 输出 | 适用场景 |
|------|------|------|------|----------|
| DLinear | 深度学习（线性） | [N, 2016, 2] | [N, 648, 1] | 所有站点，批量推理 |
| TimeXer | 深度学习（Transformer） | [N, 2016, 2] | [N, 648, 1] | 所有站点，批量推理 |
| XGBoost | 机器学习（树模型） | 特征工程后的表格数据 | [288] 或 [576] | 白名单站点，逐站推理 |

## 2. DLinear 模型

### 2.1 论文来源

[Are Transformers Effective for Time Series Forecasting?](https://arxiv.org/pdf/2205.13504.pdf) (AAAI 2023)

### 2.2 核心思想

DLinear 是一个极简的线性模型，通过**序列分解**将输入分为趋势项和季节项，分别用线性层映射到预测长度。

### 2.3 模型结构

```
输入 x: [batch, seq_len, channels]
         │
         ▼
┌─────────────────────┐
│  Series Decomposition│  (移动平均分解)
│  (moving_avg kernel) │
└─────────┬───────────┘
          │
    ┌─────┴─────┐
    ▼           ▼
┌────────┐  ┌────────┐
│Seasonal│  │ Trend  │
│ Linear │  │ Linear │
│seq→pred│  │seq→pred│
└────┬───┘  └────┬───┘
     │           │
     └─────┬─────┘
           ▼
      output = seasonal + trend
      [batch, pred_len, channels]
```

### 2.4 关键参数

```python
seq_len = 2016      # 输入序列长度（7天 × 288点）
pred_len = 648      # 预测序列长度（2.25天 × 288点）
enc_in = 1          # 输入通道数（单变量：load）
moving_avg = 25     # 分解核大小
```

### 2.5 推理流程

```python
# exp_long_term_forecasting.py → test_online()
def test_online(self, data_np):
    for batch_x in test_loader:
        # 实例标准化
        mean = batch_x.mean(dim=1, keepdim=True)
        std = batch_x.std(dim=1, keepdim=True)
        x_norm = (batch_x - mean) / (std + 1e-8)
        
        # 模型推理
        outputs = self.model(x_norm)
        
        # 反标准化
        outputs = outputs * (std + 1e-8) + mean
        outputs = outputs[:, -pred_len:, :]
```

## 3. TimeXer 模型

### 3.1 核心思想

TimeXer 是一个基于 Transformer 的时序预测模型，创新点在于：
- **Patch Embedding**：将时间序列切分为 patch，减少序列长度
- **全局 Token**：引入可学习的全局 token 捕获整体模式
- **Cross-Attention**：内生变量（load）与外生变量（时间标记）之间的交叉注意力

### 3.2 模型结构

```
输入: x_enc [batch, seq_len, n_vars]
      x_mark_enc [batch, seq_len, mark_dim]  (时间标记)
         │              │
         ▼              ▼
┌──────────────┐  ┌──────────────────┐
│ EnEmbedding  │  │ DataEmbedding_inv│
│ (Patch+Pos+  │  │ (外生变量嵌入)    │
│  Global Token)│  │                  │
└──────┬───────┘  └────────┬─────────┘
       │                   │
       ▼                   ▼
┌──────────────────────────────────┐
│         Encoder                   │
│  ┌─────────────────────────────┐ │
│  │ EncoderLayer × e_layers     │ │
│  │  - Self-Attention (patches) │ │
│  │  - Cross-Attention          │ │
│  │    (global_token ↔ exog)    │ │
│  │  - FFN (Conv1d)             │ │
│  └─────────────────────────────┘ │
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────┐
│ FlattenHead              │
│ flatten → Linear → pred  │
│ [bs, nvars, d_model,     │
│  patch_num] → [bs, nvars,│
│  pred_len]               │
└──────────────────────────┘
               │
               ▼
         output [batch, pred_len, 1]
```

### 3.3 关键组件

**EnEmbedding（内生变量嵌入）**：
```python
class EnEmbedding:
    # 将输入切分为 patch
    x = x.unfold(dimension=-1, size=patch_len, step=patch_len)
    # 线性映射到 d_model
    x = value_embedding(x) + position_embedding(x)
    # 拼接全局 token
    x = torch.cat([x, glb_token], dim=2)
```

**EncoderLayer（编码器层）**：
```python
class EncoderLayer:
    # 1. Self-Attention: patches 之间的自注意力
    x = x + self_attention(x, x, x)
    x = norm1(x)
    
    # 2. Cross-Attention: 全局 token 与外生变量的交叉注意力
    x_glb = cross_attention(x_glb, cross, cross)
    x_glb = norm2(x_glb)
    
    # 3. FFN
    y = conv2(activation(conv1(x)))
    return norm3(x + y)
```

### 3.4 关键参数

```python
seq_len = 2016       # 输入序列长度
pred_len = 648       # 预测序列长度
patch_len = 96       # Patch 大小
d_model = 512        # 模型维度
n_heads = 8          # 注意力头数
e_layers = 2         # 编码器层数
d_ff = 2048          # FFN 维度
enc_in = 2           # 输入变量数（timestamp + load）
use_norm = True      # 使用 Non-stationary Normalization
```

### 3.5 标准化策略

TimeXer 使用 Non-stationary Transformer 的标准化方法：
```python
# 编码前：减均值除标准差
means = x_enc.mean(1, keepdim=True).detach()
x_enc = x_enc - means
stdev = torch.sqrt(torch.var(x_enc, dim=1, keepdim=True, unbiased=False) + 1e-5)
x_enc /= stdev

# 解码后：反标准化
dec_out = dec_out * stdev + means
```

## 4. XGBoost 模型（LoadXGB5fold）

### 4.1 训练策略

- **5折交叉验证**：按日期间隔划分（每5天一折），非随机划分
- **贝叶斯超参优化**（可选）：使用 `bayes_opt` 库搜索最优超参数
- **Top-K 集成**：从5个模型中选择验证得分最好的 top_k=3 个模型，预测时取平均

### 4.2 超参数空间


### 4.3 训练数据量

- **白名单站点**：最近50天数据训练
- **普通站点**：最近20天数据训练
- **评估指标**：MAE（eval_metric='mae'）
- **早停**：early_stopping_rounds=30, num_boost_round=250

### 4.4 预测流程

```python
class ModelLoad:
    def predict(self, station_id, his_data, pred_date):
        # 1. 特征工程
        data = LoadFeatureEngineer().feature_engineer_v2(his_data)
        
        # 2. 构建预测特征（pred_date 当天的时间特征）
        pred_feature = data[pred_date].drop(['load', 'date', 'target'])
        
        # 3. Top-K 模型平均预测
        predictions = [model.predict(pred_feature) for model in best_models]
        output = mean(predictions)
        
        # 4. 负值截断
        output = output.clip(lower=0)
        
        return output  # [288] 或 [576]（两天）
```

## 5. 模型推理的线程安全

深度学习模型推理使用全局锁保护：

```python
# multi_model_dispatch.py
_dlinear_inference_lock = threading.Lock()
_timexer_inference_lock = threading.Lock()

def run_dlinear_wrapper(data_np, exp_dlinear):
    with _dlinear_inference_lock:
        # 固定随机种子确保可复现
        random.seed(2021)
        torch.manual_seed(2021)
        np.random.seed(2021)
        preds = exp_dlinear.test_online(data_np=data_np)
        return preds
```

XGBoost 推理使用另一个锁（`lock` 参数传入），确保 S3 下载和推理不会并发冲突。

## 6. 模型选择策略（select_best_7）

`select_best_model_hour_sum_v1_dst_wst()` 实现最优模型选择：

1. 获取过去 Backtesting_days 天的所有模型预测结果
2. 与实际负荷数据对比，计算每个模型每天的小时级 MAE
3. 选择过去7天中 MAE 最低的模型作为当天的最优输出
4. 将 select_best_7 结果写入 Redis 和 Kafka

输出格式（Protobuf）：
```protobuf
message LoadSelectBest7 {
  int32 interval = 1;         // 5（分钟）
  repeated double values = 2; // 预测值列表
}
```
