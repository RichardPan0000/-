# 系统架构总览

## 1. 项目简介

`sigen-load-prediction` 是一个**储能电站负荷预测系统**，用于预测各站点未来1-2天的用电负荷（load），预测粒度为5分钟（每天288个点）。系统采用多模型集成策略（XGBoost + DLinear + TimeXer），通过 `select_best_7` 算法自动选择最优模型输出。

## 2. 技术栈

| 组件 | 技术选型 |
|------|----------|
| 语言 | Python 3.x |
| 深度学习框架 | PyTorch |
| 机器学习 | XGBoost, LightGBM |
| 调度 | APScheduler (BlockingScheduler) |
| 数据库 | MySQL (站点配置) + StarRocks (时序数据) |
| 消息队列 | Kafka (预测结果输出) |
| 缓存 | Redis (白名单、select_best_7 结果) |
| 对象存储 | S3 (模型文件) |
| 配置中心 | Nacos (YAML 配置) |
| 天气数据 | Open-Meteo API |
| 容器化 | Docker |

## 4. 核心数据流

```
1. 数据获取
   database.py → get_batch_station_data_handler.py → load_data_his_handler.py
   
2. 数据预处理
   prepare_data_for_multi_model() → 对齐到2016点 → 缺失值填充
   
3. 模型推理
   multi_model_dispatch.py → DLinear/TimeXer (批量) + XGBoost (逐站)
   
4. 结果输出
   send_station_dlinear_result_to_kafka() → Kafka → 下游消费
   
5. 最优选择
   select_best_model_hour_sum_v1_dst_wst() → Redis
```

## 6. 运行模式

每个调度类支持两种模式：
- **mode=1**: 定时调度模式（生产环境），使用 APScheduler 的 cron 触发器
- **mode=2**: 单次运行模式（测试/手动触发）

## 7. 区域划分

系统按时区（time_area）划分区域，支持：
- **normal**: 普通站点（working_mode=1）
- **DE**: 德国站点（working_mode≠1, area_code='DE'）
- **CN**: 中国生产环境（is_cn_prod=True）

每个区域有独立的触发时间和白名单。
