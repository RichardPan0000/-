# MCP Intro：模型上下文协议入门

> 说明：这是“中文学习版”笔记，不是网页全文逐字翻译。内容以原文思想、关键概念、工程实践和学习练习为主。  
> 原文：https://modelcontextprotocol.io/docs/getting-started/intro

## 一句话理解

MCP 是一种开放协议，用来把 AI 应用连接到外部工具、数据源和工作流。

## 为什么需要 MCP

过去每个 Agent 框架都自己接工具：

```text
框架 A 的工具接口
框架 B 的工具接口
IDE 的工具接口
企业系统自己的接口
```

MCP 的目标是把这些连接方式标准化，让外部能力可以作为统一协议暴露给 AI 应用。

## 可以把 MCP 理解成

```text
AI 应用与外部能力之间的标准接口层
```

类似“USB-C”的类比：不同工具和数据源通过统一标准连接到不同 AI 客户端。

## MCP 可以暴露什么

- Tools：可执行动作。
- Resources：上下文资源。
- Prompts：可复用提示模板。

## 适合你的场景

你可以把自己的业务系统做成 MCP Server：

- 光伏 IV 数据查询。
- 储能 BMS 告警查询。
- 代码仓库检索。
- 文档知识库检索。
- 专利模板生成。
- 内部 API 调用。

这样不同 Agent / IDE / coding 工具都可以复用同一套能力。


## 建议练习

1. 用自己的业务场景复述本文核心思想。
2. 写出一个可以落地的小例子，不要只停留在概念。
3. 总结：哪些部分适合固定 workflow，哪些部分适合交给 LLM/Agent 动态决策。
