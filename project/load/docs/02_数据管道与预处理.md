# 数据管道与预处理

## 1. 数据源

### 1.1 站点配置（MySQL）

表：`sigen_ai.station_config_cdc`

| 字段 | 说明 |
|------|------|
| station_id | 站点唯一标识 |
| latitude / longitude | 经纬度 |
| feature_signature_a | 特征签名（决定数据获取方式） |
| time_zone_name | 站点所在时区 |
| working_mode | 工作模式（1=普通, 其他=特殊） |
| area_code | 区域代码（如 'DE'） |

### 1.2 历史负荷数据（StarRocks）

表：`sigen_device.station_statistics_min` / `station_statistics_hour`

- 5分钟粒度的站点统计数据
- 关键字段：`statistics_time`（Unix时间戳秒）、`pcs_output_total`、`pcs_input_total`、`elec_buy_total`、`elec_sold_total`、`filtered_load_power`

### 1.3 负荷计算公式

**非绿电站点（普通站点）：**
```python
load = pcs_output_total + elec_buy_total - pcs_input_total - elec_sold_total
```

**绿电站点（feature_a 标记）：**
```python
load = filtered_load_power * 5 / 60 * 100
# filtered_load_power 是经过滤波处理的负荷功率
# * 5/60 将功率转换为5分钟能量
# * 100 归一化系数
```

### 1.4 绿电站点判断

通过 `feature_signature_a` 的二进制位判断：
```python
def check_station_feature_a(feature_a):
    # 检查二进制第3位或第5位是否为1
    binary = format(int(feature_a), '016b')
    return binary[-3] == '1' or binary[-5] == '1'
```

## 2. 数据获取流程

### 2.1 批量获取（get_batch_station_data_handler.py）

核心函数：`get_area_station_data_his_generator()`

```
输入: area(时区), batch_size, end_date, past_days_num
输出: Generator[(batch_df, station_mode_list)]

流程:
1. get_area_station_list(area) → 获取该时区所有AI站点
2. 按 feature_a 分为绿电/非绿电两组
3. 按 batch_size 分批
4. 每批调用对应的数据获取函数
5. 以生成器方式逐批返回 (DataFrame, station_list)
```

生成器设计的优势：避免一次性加载所有站点数据到内存。

## 3. 数据预处理

### 3.1 多模型输入准备（prepare_data_for_multi_model）

位于 `load_pred_schedule.py`，将原始 DataFrame 转换为深度学习模型所需的 numpy 数组。

**输入**: DataFrame [station_id, statistics_time, load]
**输出**: numpy array [N_stations, 2016, 2]（最后一维为 [timestamp, load]）

**处理步骤**：

1. **时间对齐**：生成完整的5分钟时间轴（从 min_time 到 max_time）
2. **截取2016点**：只保留最新的2016个时间点（= 7天 × 288点/天）
3. **缺失值填充**：
   - 如果有 ≥288 个有效点：使用**同时刻平均值**填充（利用日内周期性）
   - 如果 <288 个有效点：使用线性插值
   - 兜底：前向填充 → 后向填充 → 0填充
4. **不足2016点时**：在前面用第一个值补齐

### 3.2 DLinear/TimeXer 输入处理

```python
def process_batch_for_dlinear(batch_df, exp_dlinear):
```

### 3.3 XGBoost 输入处理

XGBoost 使用特征工程后的数据，流程不同：

```
1. 获取站点历史数据（8-13天）
2. FillDataHandler 补齐缺失时间点
3. LoadFeatureEngineer.feature_engineer_v2() 生成特征
4. 截取训练窗口（白名单站点50天，普通站点20天）
5. 5折交叉验证训练
```

## 4. 特征工程（XGBoost 专用）

`LoadFeatureEngineer.feature_engineer_v2()` 生成的特征包括：

- **时间特征**：hour, minute, day_of_week, month, is_weekend
- **滞后特征**：load 的 shift(1天, 2天, 7天)
- **统计特征**：rolling mean/std/max/min（窗口3/6/12/24/48）
- **差分特征**：diff(1), diff(7天)
- **目标编码**：按小时/星期的历史均值
