## ✅ 一、核心结论（先看表）

| 任务类型           | 输出神经元数 | 激活函数                             | 损失函数                                                   | 是否需要 softmax/sigmoid？                           |
| ------------------ | ------------ | ------------------------------------ | ---------------------------------------------------------- | ---------------------------------------------------- |
| **二分类**         | 1 或 2       | **Sigmoid**（1输出）或 不用（2输出） | `BCEWithLogitsLoss`（1输出）或 `CrossEntropyLoss`（2输出） | ✅ 1输出时需 sigmoid（隐含在 `BCEWithLogitsLoss` 中） |
| **多分类（互斥）** | = 类别数     | **Softmax**（隐含）                  | `CrossEntropyLoss`                                         | ✅ 但通常不显式写，损失函数内部处理                   |
| **多标签分类**     | = 标签数     | **Sigmoid**                          | `BCEWithLogitsLoss`                                        | ✅ 每个输出独立用 sigmoid                             |





在**二分类（binary classification）\**和\**多分类（multi-class classification）\**任务中，`softmax`、`sigmoid`、`CrossEntropyLoss`、`BCEWithLogitsLoss` 等函数和loss的使用，是\**成套搭配**的，不然会出问题。下面我给你系统性地总结。

------

### ✅ 一、二分类任务

#### 1. 网络最后一层输出：

- 一般为 **单个神经元输出一个实数**（logit）
- 不需要手动加 `sigmoid`，除非你手动做预测

#### 2. 使用的激活函数：

- **训练时：不要加 `sigmoid`**，用 `BCEWithLogitsLoss`（它内部自动做了 `sigmoid` 和 `log loss`）
- **预测时（inference）：才加 sigmoid** 得到概率

#### 3. 常用 Loss：

- ✅ `BCEWithLogitsLoss` （推荐）
- 也可以：`BCELoss` + `sigmoid`（不推荐，因为数值稳定性差）

#### 4. 标签格式：

- 标签为 `0` 或 `1` 的浮点型（`torch.float`，不能是 long）

#### 举个例子：

```python
# 模型输出一个logit
logits = model(x)  # shape: [batch_size, 1]

# loss函数
loss_fn = torch.nn.BCEWithLogitsLoss()
loss = loss_fn(logits, y.float())  # y: shape [batch_size, 1]
```

------

### ✅ 二、多分类任务（单标签）

#### 1. 网络最后一层输出：

- 输出为 **类别数（num_classes）个logits**，即 `shape = [batch_size, num_classes]`

#### 2. 使用的激活函数：

- **训练时不要加 `softmax`**，使用 `CrossEntropyLoss`（内部自带 `softmax + log`）
- **预测时再加 `softmax`** 或 `argmax` 做分类判断

#### 3. 常用 Loss：

- ✅ `CrossEntropyLoss`（推荐）

#### 4. 标签格式：

- 标签为 `整数 class_id`，形状为 `[batch_size]`，数据类型为 `torch.long`

#### 举个例子：

```python
# 模型输出多个logits
logits = model(x)  # shape: [batch_size, num_classes]

# loss函数
loss_fn = torch.nn.CrossEntropyLoss()
loss = loss_fn(logits, y)  # y: shape [batch_size]，值为 0~num_classes-1
```

------

### ✅ 三、多标签分类（每个样本可以属于多个类）

#### 用法类似二分类，只不过是对每个类都做一个二分类。

#### 1. 网络输出：

- 输出维度为 `[batch_size, num_classes]`，每个位置是一个logit

#### 2. Loss：

- ✅ `BCEWithLogitsLoss`

#### 3. 标签格式：

- 标签是 `0` 或 `1` 组成的 multi-hot 向量，形状为 `[batch_size, num_classes]`

#### 举个例子：

```python
# 模型输出多个logits
logits = model(x)  # shape: [batch_size, num_classes]

# 多标签多热编码 y
loss_fn = torch.nn.BCEWithLogitsLoss()
loss = loss_fn(logits, y.float())
```

------

### ✅ 总结：激活函数、loss、标签配套表

| 任务类型   | 输出维度          | Loss 函数           | 激活函数（训练） | 激活函数（预测） | 标签类型                    |
| ---------- | ----------------- | ------------------- | ---------------- | ---------------- | --------------------------- |
| 二分类     | `[B, 1]` or `[B]` | `BCEWithLogitsLoss` | ❌ 无             | ✅ `sigmoid`      | `[B]`, 值为 0 或 1, float   |
| 多分类     | `[B, C]`          | `CrossEntropyLoss`  | ❌ 无             | ✅ `softmax`      | `[B]`, 值为 0~C-1, long     |
| 多标签分类 | `[B, C]`          | `BCEWithLogitsLoss` | ❌ 无             | ✅ `sigmoid`      | `[B, C]`, multi-hot float型 |

------

### ❗常见错误

- ❌ `CrossEntropyLoss` 前面用了 `softmax` —— 会导致梯度消失或不收敛
- ❌ `BCEWithLogitsLoss` 前面加了 `sigmoid` —— 会重复计算 sigmoid，影响稳定性
- ❌ 标签数据类型错误，如 `CrossEntropyLoss` 用了 `float` 类型标签，或 `BCEWithLogitsLoss` 用了 `long` 类型
