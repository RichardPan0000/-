# 一致性算法与方法总结

## 资料来源

- `热失控、一致性相关的专利总结提示词.txt`
- `热失控、一致性相关的专利总结提示词2(补充专利).txt`

本文只总结两份文本中与“电池/储能系统一致性”直接相关，或可用于一致性评估、筛选、预测、异常识别、均衡控制的方法。纯热失控预警、消防、气体传感、结构防护等内容未纳入；如果某项热失控方法使用了跨电芯差异、温差、SOC/SOH偏离等一致性特征，则按相关程度纳入分析。

## 总体结论

两份材料中的一致性方法主要集中在四类：

1. **电压/SOC/电量差异评估**：数量最多，主要通过电压差、SOC区间、末端电压、放电时序标准化、V-Q曲线等方式识别单体电量或电压不一致。
2. **异常单体筛查与风险累积**：通过电压偏差、内阻离散、离群趋势、历史风险累计、异常单体比例等方式发现一致性劣化。
3. **统计相关性与多源融合评估**：包括高低频时空特征融合、完整充放电统计差异、协方差矩阵一致性系数等。
4. **温度一致性与均衡控制**：温差控制方法较多，但真正涉及闭环均衡控制的方法较少，主要代表是单神经元自学习均衡控制。

整体看，现有专利更偏“检测/评估/预警”，较少覆盖“检测结果到均衡策略”的闭环决策。对于算法部门而言，最值得关注的是时间序列异常检测、工况归一化、多源特征融合、风险评分与置信度建模。

| 19 | 统计相关性 | 电压协方差矩阵一致性系数 | 计算单体电压协方差矩阵，归一化后用相对协方差均值作为一致性系数 | 单独工况下各电芯电压 | 各电芯一致性系数 | 在线/测试工况一致性评估 | 只看电压，工况漂移会影响协方差 |
| 20 | 自放电一致性 | SOC日差异序列 + 差异值 + 置信度 | 计算每日SOC差异，形成时间序列，再用多模型输出自放电一致性差异值和置信度 | 多日SOC值、SOC差异序列、日期序列 | 自放电差异值、置信度、异常判定 | 长期静置、低负载监控、自放电异常检测 | 需要多日数据，响应不够即时 |
| 21 | 机器学习预测 | 回归森林电压状态序列 | 用多时间段属性参数预测电压，生成状态序列后评估电压一致性 | 多时段属性参数、回归森林预测电压 | 状态序列、电压一致性结果 | 模组电压一致性评估 | 泛化依赖训练数据覆盖 |

## 可合并的技术方向

### 1. 电压-SOC曲线一致性评估

可合并方法：

- 电量不一致性电芯对识别
- 最小电芯电压-SOC一致性模型
- 电压差-SOC区间/SOC偏差
- 放电时序转置 + SOC标准化 + 电压指标
- 电压-容量曲线特征峰一致性筛选
- 回归森林电压状态序列

这些方法都围绕“电压-容量-SOC关系”来识别单体间差异。本质是把不同电芯放到相同SOC、相同电流、相同容量区间或相似工况下比较，减少运行工况对电压差的干扰。

算法上可进一步发展为：

- 工况归一化后的电压曲线嵌入表示；
- 电压-SOC曲线的形状相似度、峰值偏移、局部斜率差异；
- 基于Transformer、TCN、Shapelet或DTW的曲线一致性评分；
- 对不完整充放电片段进行片段级一致性估计。

### 2. 异常单体筛查与风险累积

可合并方法：

- 电压偏差 + 内阻离散异常分析
- 单体状态参数离群趋势
- 历史指标特征 + 一致性模型 + 动态风险累积
- 异常簇比例 + 异常单体比例

这些方法都是先识别异常单体、异常簇或异常趋势，再输出一致性风险。相比静态阈值，时间窗口风险累积更适合真实储能场景，因为它能降低瞬时噪声导致的误报。

算法上可进一步发展为：

- 多变量时间序列异常检测；
- 单体-模组-簇-柜多层级异常传播建模；
- 异常持续时间、异常幅度、异常复现频次联合评分；
- 输出带置信度的一致性风险等级。

### 3. 统计相关性一致性评价

可合并方法：

- 完整充放电统计差异评价
- 电压协方差矩阵一致性系数
- 多源时空融合一致性评估

这类方法更关注群体结构：一个单体不是单独被判断，而是看它与其他单体的相关性、同步性和偏离程度。

算法上可进一步发展为：

- 单体间相关性图谱；
- 基于图神经网络的电芯一致性评价；
- 协方差、相关系数、距离矩阵、聚类结构联合建模；
- 使用图上的离群节点识别异常单体。

### 4. 温度一致性与热均衡

可合并方法：

- 最大/最小电芯温度差控制
- 环境温度 + 电芯温差控制
- SOC/SOH/SOT综合状态预测中的温度状态预测

这类方法和热失控预警高度相关，但只有在关注电芯间温差、温度离散、热均衡时，才属于一致性方法。单纯的温度超限报警不应归为一致性算法。

算法上可进一步发展为：

- 温度场时空异常检测；
- 温差、升温速率、散热滞后联合建模；
- 温度一致性与SOC/SOH差异联动；
- 热异常单体与电压异常单体的交叉验证。

### 5. 均衡控制

当前材料中真正涉及均衡控制的方法较少，主要是单神经元自学习均衡控制。其思路是基于压差计算均衡时间和占空比，属于从检测到控制的闭环方法。

现有不足：

- 多数方法只做到检测，没有给出均衡策略；
- 均衡目标多为电压一致，较少考虑容量、SOH、内阻、自放电差异；
- 缺少“异常风险评分如何转化为均衡动作”的决策逻辑。

算法上可进一步发展为：

- 一致性评分驱动的均衡优先级排序；
- 基于风险收益的均衡动作选择；
- 将电压、SOC、SOH、温度、自放电风险统一为均衡目标函数；
- 结合约束优化或强化学习生成均衡策略。

|---:|---|---|---|
| 1 | 多源时序一致性异常检测 | 最贴合时间序列和异常检测能力 | 多变量特征融合、窗口级风险累积、置信度输出 |
| 2 | 工况归一化后的电压/SOC曲线一致性 | 现有专利多但仍有改进空间 | 不完整周期建模、曲线片段匹配、SOC标准化、形状相似度 |

## 现有专利方法的主要不足

1. **阈值规则较多，智能建模不足**  
   很多方法仍依赖电压差、温差、内阻离散度等阈值，容易受工况、温度、电流变化影响。

2. **检测多，控制少**  
   大部分方法能发现不一致，但没有进一步说明如何均衡、如何排序、如何闭环控制。

3. **单指标方法较多，多源融合不足**  
   电压、SOC、温度、SOH、内阻、自放电往往被单独建模，缺少统一的一致性风险评分。

4. **对真实储能工况适配不足**  
   完整充放电、静置、稳定电流等条件在真实储能电站中不一定常见，不完整周期和动态负载场景需要更强算法适配。

5. **缺少置信度和误报抑制机制**  
   一致性异常往往是慢变量劣化，单点报警容易误报。更合理的是输出风险趋势、置信度和持续性证据。

## 可考虑的专利研发路线

### 路线一：基于工况归一化的电压-SOC曲线一致性异常检测

核心创新点：

- 将不同电芯的运行片段统一映射到可比较的SOC、电流、温度或容量坐标系；
- 对不完整充放电片段进行曲线特征提取；
- 通过曲线形状相似度、局部斜率差异、峰值偏移、标准化电压残差输出一致性评分。

简要实现：

1. 从BMS采集电芯电压、电流、SOC、温度、时间戳；
2. 筛选可比工况片段，完成SOC/温度/电流归一化；
3. 提取曲线特征或训练时序编码模型；
4. 对每个电芯计算相对群体的一致性偏离分数；
5. 输出异常电芯、异常区间和置信度。

### 路线二：多层级一致性风险累积与异常单体定位

核心创新点：

- 将单体、模组、簇、柜建成层级结构；
- 对每个层级计算一致性风险；
- 通过时间窗口累计异常幅度、持续时间和复现频次；
- 避免单点阈值误报。

简要实现：

1. 对电压、温度、SOC、SOH、内阻等特征建立单体级异常分数；
2. 按模组、簇、柜聚合为层级风险；
3. 引入窗口累计、趋势斜率、异常复现次数；
4. 输出异常单体排名、异常簇比例、系统一致性健康分；
5. 为运维或均衡策略提供优先级。

### 路线三：一致性评分驱动的均衡控制策略

核心创新点：

- 不只根据瞬时压差均衡，而是根据综合一致性评分决定均衡对象和均衡强度；
- 将电压、SOC、SOH、温度、自放电风险统一到多目标均衡决策中；
- 输出均衡优先级、均衡时长、占空比或禁止均衡条件。

简要实现：

1. 计算每个电芯的一致性风险评分；
2. 识别造成不一致的主因：SOC偏差、容量衰减、内阻异常、自放电、温度异常；
3. 根据主因选择均衡策略；
4. 在安全约束下计算均衡时长和占空比；
5. 记录均衡前后风险变化，用于策略自学习。

## 建议筛选口径

后续继续整理专利时，建议把“一致性”分为以下几种，避免概念混乱：

| 类型 | 是否建议纳入算法总结 | 说明 |
|---|---|---|
| 电芯性能一致性 | 是 | 电压、SOC、容量、SOH、内阻、自放电等跨电芯差异 |
| 温度一致性 | 条件纳入 | 只有涉及温差、热均衡、温度离散时纳入 |
| 系统配置参数一致性 | 低优先级 | 偏运维和配置检查，算法价值较低 |
| 诊断结果一致性 | 条件纳入 | 端云诊断一致、模型结果一致可作为辅助机制 |
| 单纯热失控报警 | 不纳入 | 若没有跨电芯差异或一致性特征，不属于一致性算法 |
| 均衡控制 | 高优先级 | 检测到控制闭环，专利价值较高 |

## 最终判断

两份文本中的一致性算法可以概括为：

- **基础层**：电压差、温差、SOC差、内阻离散度、容量曲线差异；
- **评估层**：统计差异、协方差、曲线特征、时空融合特征；
- **预测层**：离群趋势、历史风险累积、回归森林、电压/SOC序列标准化；
- **系统层**：异常单体比例、异常簇比例、站级健康评分；
- **控制层**：均衡时间、占空比、均衡优先级。

从算法人员视角，最值得深挖的是：**多源时序异常检测 + 工况归一化曲线一致性 + 层级风险累积 + 均衡闭环决策**。这几个方向既贴合现有专利脉络，又能避开单纯硬件、消防、嵌入式流程类方案。
